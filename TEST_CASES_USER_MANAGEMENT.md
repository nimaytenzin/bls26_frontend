# Test Cases - User Management Module

## Test Case Documentation

| Test ID | Module / Feature | Objective | Expected Results | Status (P/F/B) |
|---------|------------------|-----------|------------------|----------------|
| TC1-201 | Load Users | Verify that all users (Admins, Supervisors, Enumerators) are loaded and displayed correctly | 1. User Management page loads successfully <br> 2. Three tabs display: "Admins", "Supervisors", and "Enumerators" <br> 3. All users are loaded and displayed in respective tables <br> 4. Loading indicators show during data fetch <br> 5. Statistics cards display correct counts for each role <br> 6. Error messages display if loading fails | P |
| TC1-202 | View User Profile | Verify that admin can view complete user profile information for any user | 1. Clicking "View Profile" button opens profile view dialog <br> 2. Dialog header displays: "View [Role] Profile" <br> 3. Profile displays user basic information: Name, CID, Email, Phone, Role, Created Date <br> 4. For Supervisors: displays assigned dzongkhags <br> 5. For Enumerators: displays survey assignments (active and ended) <br> 6. Profile data loads correctly from API <br> 7. Error message displays if profile fails to load | P |
| TC1-203 | Create Admin User | Verify that admin can create a new admin user successfully | 1. "Add Admin" button opens "Create New Admin" dialog <br> 2. Dialog displays form fields: Full Name, CID, Email Address, Phone Number, Password, Confirm Password, Role <br> 3. Form validation works: required fields, email format, CID (11 digits), phone (8 digits), password (min 6 chars), password match <br> 4. Submitting valid form creates admin user <br> 5. Success message displays: "Admin created successfully" <br> 6. Dialog closes and Admins table refreshes with new user <br> 7. Error messages display for validation failures or server errors | P |
| TC1-204 | Edit Admin User | Verify that admin can update admin user information | 1. Clicking "Edit" button on admin row opens "Edit Admin" dialog <br> 2. Dialog pre-fills with current user data: Name, CID (disabled), Email, Phone, Role <br> 3. CID field is disabled and cannot be edited <br> 4. Form validation works for editable fields <br> 5. Submitting with changes updates user successfully <br> 6. Success message displays: "Admin updated successfully" <br> 7. Dialog closes and table refreshes with updated data <br> 8. If no changes made, info message: "No changes to save" <br> 9. Error messages display for validation failures or server errors | P |
| TC1-205 | View Admin Profile | Verify that admin can view admin user profile details | 1. Clicking "View Profile" button on admin row opens "View Admin Profile" dialog <br> 2. Dialog displays admin information: Name, CID, Email, Phone, Role, Created Date <br> 3. Profile data loads correctly from API <br> 4. Dialog is maximizable <br> 5. Error message displays if profile fails to load | P |
| TC1-206 | Create Supervisor User | Verify that admin can create a new supervisor user successfully | 1. "Add Supervisor" button opens "Create New Supervisor" dialog <br> 2. Form validation works: required fields, email format, CID (11 digits), phone (8 digits), password (min 6 chars), password match <br> 4. Submitting valid form creates supervisor user <br> 5. Success message displays: "Supervisor created successfully" <br> 6. Dialog closes and Supervisors table refreshes with new user <br> 7. Error messages display for validation failures or server errors | P |
| TC1-207 | Edit Supervisor User | Verify that admin can update supervisor user information | 1. Clicking "Edit" button on supervisor row opens "Edit Supervisor" dialog <br> 2. Dialog pre-fills with current user data: Name, CID (disabled), Email, Phone <br> 3. CID field is disabled and cannot be edited <br> 4. Assigned dzongkhags are displayed (read-only) <br> 5. Form validation works for editable fields <br> 6. Submitting with changes updates user successfully <br> 7. Success message displays: "Supervisor updated successfully" <br> 8. Dialog closes and table refreshes with updated data <br> 9. Error messages display for validation failures or server errors | P |
| TC1-208 | View Supervisor Profile | Verify that admin can view supervisor user profile details | 1. Clicking "View Profile" button on supervisor row opens "View Supervisor Profile" dialog <br> 2. Dialog displays supervisor information: Name, CID, Email, Phone, Role, Created Date <br> 3. Assigned dzongkhags are displayed <br> 4. Profile data loads correctly from API <br> 5. Dialog is maximizable <br> 6. Error message displays if profile fails to load | P |
| TC1-209 | Assign Dzongkhags to Supervisor | Verify that admin can assign dzongkhags to a supervisor | 1. Clicking "Assign Dzongkhags" button on supervisor row opens "Manage Dzongkhag Assignments" dialog <br> 2. Dialog displays all available dzongkhags in a multi-select interface <br> 3. Currently assigned dzongkhags are pre-selected <br> 4. Admin can select/deselect dzongkhags <br> 5. Clicking "Save" updates assignments <br> 6. Success message displays: "Dzongkhag assignments updated successfully" <br> 7. Dialog closes and Supervisors table refreshes <br> 8. Assigned dzongkhags are displayed in supervisor row <br> 9. If no changes made, info message: "No changes to save" <br> 10. Error messages display if save fails | P |
| TC1-210 | Reset User Password | Verify that admin can reset any user's password | 1. Clicking "Reset Password" button opens "Reset Password" dialog <br> 2. Dialog displays user name and form fields: New Password, Confirm Password <br> 3. Password validation: minimum 8 characters, must contain at least 3 of: uppercase, lowercase, number, special character <br> 4. Password strength indicator shows weak/medium/strong <br> 5. Password match validation works <br> 6. Submitting valid password resets user password successfully <br> 7. Success message displays: "Password reset successfully" <br> 8. Dialog closes <br> 9. User can login with new password <br> 10. Error messages display for validation failures or server errors | P |

## Test Status Legend
- **P** = Pass (Test case is ready for execution)
- **F** = Fail (Test case needs revision or has issues)
- **B** = Blocked (Test case cannot be executed due to dependencies)
