# Test Cases - Enumeration Area Merge Operation

## Test Case Documentation

| Test ID | Module / Feature | Objective | Expected Results | Status (P/F/B) |
|---------|------------------|-----------|------------------|----------------|
| TC1-701 | Load Merge Operation Page | Verify that merge operation page loads correctly with location filters | 1. Merge Enumeration Areas page loads successfully <br> 2. Location selection dropdowns are visible: Dzongkhag, Administrative Zone, Sub-Administrative Zone <br> 3. All dzongkhags are loaded in dropdown <br> 4. Form fields for merged EA are visible: Name, Area Code, Description, GeoJSON File <br> 5. Selected EAs container is visible <br> 6. Loading indicators show during data fetch | P |
| TC1-702 | Select Location for Browsing EAs | Verify that selecting location filters and loads enumeration areas correctly | 1. Selecting dzongkhag loads corresponding administrative zones <br> 2. Selecting administrative zone loads corresponding sub-administrative zones <br> 3. Selecting sub-administrative zone loads enumeration areas from that SAZ <br> 4. EA dropdown displays active enumeration areas from selected SAZ <br> 5. Selected EAs container persists when changing location filters <br> 6. Error messages display if location loading fails | P |
| TC1-703 | Select Enumeration Areas for Merge | Verify that admin can select multiple enumeration areas from different SAZs | 1. EA dropdown displays enumeration areas from selected SAZ <br> 2. Selecting an EA from dropdown adds it to selected EAs container <br> 3. Selected EAs are displayed in a list with name, area code, and SAZ information <br> 4. Admin can switch to different SAZ and add more EAs <br> 5. Selected EAs from multiple SAZs are displayed together <br> 6. Remove button allows removing individual EAs from selection <br> 7. Validation ensures at least 2 EAs are selected before merge | P |
| TC1-704 | Create Merged Enumeration Area | Verify that admin can create a merged enumeration area successfully | 1. Form displays fields: Name (required, min 2 chars), Area Code (required, alphanumeric), Description (required, min 5 chars), GeoJSON File (required) <br> 2. Form validation works for all required fields and formats <br> 3. GeoJSON file upload accepts .json and .geojson files <br> 4. Sub-administrative zone IDs are auto-collected from selected source EAs <br> 5. Submitting valid form creates merged EA <br> 6. Success message displays: "Enumeration areas merged successfully" <br> 7. Source EAs are deactivated and merged EA is created <br> 8. Error messages display for validation failures or server errors | P |
| TC1-705 | View List of Merged EAs | Verify that admin can view paginated list of merged enumeration areas | 1. Merged EAs list page loads successfully <br> 2. Table displays merged EAs with columns: ID, Name, Area Code, Description, Operation Date, Actions <br> 3. Pagination controls are functional (10, 25, 50, 100 rows per page) <br> 4. Table supports sorting by operation date (default: descending) <br> 5. Search functionality filters merged EAs by name, area code, or description <br> 6. Loading indicator shows during data fetch <br> 7. Error message displays if loading fails | P |
| TC1-706 | View Merge History | Verify that admin can view detailed history of merge operation | 1. "View History" button is visible for each merged EA in the list <br> 2. Clicking "View History" opens history viewer dialog <br> 3. History displays: source EAs that were merged, merged EA details, operation date, reason (if provided) <br> 4. History information is accurate and complete <br> 5. Dialog can be closed or maximized <br> 6. Error message displays if history loading fails | P |
| TC1-707 | Search Merged EAs | Verify that search functionality filters merged enumeration areas correctly | 1. Search input field accepts text input <br> 2. Search filters results by name, area code, or description <br> 3. Clear search button appears when search has value <br> 4. Clearing search resets table to show all merged EAs <br> 5. Search is case-insensitive and supports partial matches <br> 6. Search works with pagination | P |

## Test Status Legend
- **P** = Pass (Test case is ready for execution)
- **F** = Fail (Test case needs revision or has issues)
- **B** = Blocked (Test case cannot be executed due to dependencies)
