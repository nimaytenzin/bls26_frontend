# Test Cases - Enumeration Area Management Module

## Test Case Documentation

| Test ID | Module / Feature | Objective | Expected Results | Status (P/F/B) |
|---------|------------------|-----------|------------------|----------------|
| TC1-601 | Load Enumeration Areas | Verify that enumeration areas are loaded and displayed correctly based on selected filters | 1. Enumeration Area Management page loads successfully <br> 2. Dzongkhag dropdown displays all available dzongkhags <br> 3. Administrative Zone dropdown displays zones for selected dzongkhag <br> 4. Sub-Administrative Zone dropdown displays zones for selected administrative zone <br> 5. First dzongkhag is auto-selected by default <br> 6. Enumeration areas table displays areas for selected filters with columns: ID, Name, Description, Area Code, Sub-Administrative Zones, Actions <br> 7. Map displays enumeration area boundaries <br> 8. Loading indicator shows during data fetch <br> 9. Error message displays if loading fails | P |
| TC1-602 | Select Filters (Dzongkhag, Administrative Zone, Sub-Administrative Zone) | Verify that selecting filters updates and loads corresponding enumeration areas | 1. Dzongkhag dropdown is visible and functional <br> 2. Administrative Zone dropdown updates based on selected dzongkhag <br> 3. Sub-Administrative Zone dropdown updates based on selected administrative zone <br> 4. Selecting different filters updates the enumeration areas list <br> 5. Map refreshes to show boundaries for selected filters <br> 6. Selected filters are saved and persist <br> 7. Enumeration area selection is reset when filters change | P |
| TC1-603 | View Enumeration Area | Verify that admin can view enumeration area details and data | 1. Clicking "View Details" button navigates to enumeration area data viewer page <br> 2. Clicking on map feature displays popup with area information (name, description, area code) <br> 3. Selecting area from table highlights it in the map <br> 4. Map popup shows download options (GeoJSON, KML) <br> 5. Area labels are displayed on map with name and area code | P |
| TC1-604 | Create Enumeration Area | Verify that admin can create a new enumeration area successfully | 1. "Add Enumeration Area" button opens "Add New Enumeration Area" dialog <br> 2. Dialog displays form fields: Name (required, min 2 chars), Description (required), Area Code (required, alphanumeric), Sub-Administrative Zones (required, multi-select, min 1) <br> 3. Form validation works for required fields and format <br> 4. Submitting valid form creates enumeration area <br> 5. Success message displays: "Enumeration area created successfully" <br> 6. Dialog closes and table refreshes with new area <br> 7. Map refreshes to show new area boundary <br> 8. Error messages display for validation failures or server errors | P |
| TC1-605 | Edit Enumeration Area | Verify that admin can update enumeration area information | 1. Clicking "Edit" button on area row opens "Edit Enumeration Area" dialog <br> 2. Dialog pre-fills with current area data: Name, Description, Area Code, Sub-Administrative Zones <br> 3. Form validation works for editable fields <br> 4. Submitting with changes updates area successfully <br> 5. Success message displays: "Enumeration area updated successfully" <br> 6. Dialog closes and table refreshes with updated data <br> 7. Map refreshes to reflect changes <br> 8. Error messages display for validation failures or server errors | P |
| TC1-606 | Delete Enumeration Area | Verify that admin can delete an enumeration area | 1. Clicking "Delete" button on area row opens confirmation dialog <br> 2. Confirmation dialog displays: "Are you sure you want to delete [Name]?" with warning message <br> 3. Confirmation dialog has "Cancel" and "Delete" buttons <br> 4. Clicking "Delete" removes enumeration area <br> 5. Success message displays: "Enumeration area deleted successfully" <br> 6. Area is removed from table and map <br> 7. Clicking "Cancel" closes dialog without deletion <br> 8. Error messages display if deletion fails | P |
| TC1-607 | Upload GeoJSON | Verify that admin can upload GeoJSON file for an enumeration area | 1. Clicking "Upload GeoJSON" button opens "Upload GeoJSON" dialog <br> 2. Dialog displays selected area information and file upload field <br> 3. File upload accepts .json and .geojson files (max 10MB) <br> 4. File validation works: shows error for invalid file types <br> 5. Submitting valid file uploads GeoJSON successfully <br> 6. Success message displays: "GeoJSON uploaded successfully" <br> 7. Dialog closes and map refreshes with new boundaries <br> 8. Error messages display for invalid files or upload failures | P |
| TC1-608 | Bulk Upload Enumeration Areas | Verify that admin can bulk upload multiple enumeration areas via CSV/Excel file | 1. "Bulk Upload" button opens bulk upload dialog <br> 2. Dialog requires sub-administrative zone selection and file upload <br> 3. File upload accepts supported formats (CSV/Excel) <br> 4. Upload process shows progress indicator <br> 5. Upload results display: success count, failure count, and error details <br> 6. Successfully uploaded areas appear in table <br> 7. Error messages display for invalid files or upload failures | P |
| TC1-609 | Search Enumeration Areas | Verify that search functionality filters areas correctly | 1. Search input field accepts text input <br> 2. Search filters results by ID, name, description, or area code <br> 3. Clear search button (X) appears when search has value <br> 4. Clearing search resets table to show all areas for selected filters <br> 5. Search is case-insensitive and supports partial matches | P |
| TC1-610 | Hierarchical Table View | Verify that hierarchical table view displays enumeration areas grouped by administrative structure | 1. Hierarchical table view option is available <br> 2. Table displays enumeration areas grouped by Dzongkhag > Administrative Zone > Sub-Administrative Zone <br> 3. Expandable rows allow viewing enumeration areas under each sub-administrative zone <br> 4. Hierarchical structure is clearly visible with indentation or grouping <br> 5. Actions are available for each enumeration area in hierarchical view | P |
| TC1-611 | Map View | Verify that map view displays correctly with enumeration area boundaries | 1. Map loads and displays in the right panel <br> 2. Enumeration area boundaries are rendered on the map with proper colors <br> 3. Map is centered on selected filters with appropriate zoom level <br> 4. Area labels display name and area code on map <br> 5. Loading overlay displays while map data is being fetched <br> 6. Map container is properly sized and responsive | P |
| TC1-612 | Download GeoJSON from Map | Verify that admin can download GeoJSON file for an enumeration area from map popup | 1. Clicking on an enumeration area feature on the map opens a popup <br> 2. Popup displays area name, description, area code, and download buttons <br> 3. Clicking "GeoJSON" button in popup downloads the feature as GeoJSON file <br> 4. Downloaded file has format: "[area_name].geojson" <br> 5. Success message displays: "GeoJSON file downloaded successfully" <br> 6. File contains FeatureCollection with the selected area feature | P |
| TC1-613 | Download KML from Map | Verify that admin can download KML file for an enumeration area from map popup | 1. Clicking on an enumeration area feature on the map opens a popup <br> 2. Success message displays: "KML file downloaded successfully" <br> 6. File contains KML format with the selected area feature | P |
| TC1-614 | Basemap Selection | Verify that basemap selector allows switching between different map styles | 1. Basemap selector dropdown is visible in map header <br> 2. Area boundaries remain visible after basemap change <br> 5. Selected basemap persists during the session <br> 6. Default basemap is "Positron" | P |
| TC1-615 | Layer Toggles | Verify that layer toggles show/hide different map layers correctly | 1. Layer toggle control is visible (Sub-Administrative Zones) <br> 2. | TC1-616 | Map Interaction | Verify that map interactions work correctly | 1. Clicking on map feature selects corresponding area in table <br> 2. ## Test Status Legend
- **P** = Pass (Test case is ready for execution)
- **F** = Fail (Test case needs revision or has issues)
- **B** = Blocked (Test case cannot be executed due to dependencies)
