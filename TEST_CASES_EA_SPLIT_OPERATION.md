# Test Cases - Enumeration Area Split Operation

## Test Case Documentation

| Test ID | Module / Feature | Objective | Expected Results | Status (P/F/B) |
|---------|------------------|-----------|------------------|----------------|
| TC1-801 | Load Split Operation Page | Verify that split operation page loads correctly with location filters | 1. Split Enumeration Areas page loads successfully <br> 2. Location selection dropdowns are visible: Dzongkhag, Administrative Zone, Sub-Administrative Zone <br> 3. All dzongkhags are loaded in dropdown <br> 4. Source EA selection dropdown is visible <br> 5. Form fields for new EAs are visible (minimum 2 required) <br> 6. Add/Remove EA buttons are functional <br> 7. Loading indicators show during data fetch | P |
| TC1-802 | Select Location and Source EA | Verify that selecting location and source EA loads correctly | 1. Selecting dzongkhag loads corresponding administrative zones <br> 2. Selecting administrative zone loads corresponding sub-administrative zones <br> 3. Selecting sub-administrative zone loads enumeration areas from that SAZ <br> 4. Source EA dropdown displays active enumeration areas from selected SAZ <br> 5. Selecting source EA displays its details (name, area code, description, SAZs) <br> 6. Source EA's sub-administrative zones are auto-populated for new EAs <br> 7. Error messages display if location or EA loading fails | P |
| TC1-803 | Add New EA Forms for Split | Verify that admin can add multiple new EA forms for split operation | 1. Initial form displays with 2 new EA forms (minimum required) <br> 2. "Add New EA" button adds additional EA form to the list <br> 3. Each new EA form has fields: Name (required), Area Code (required), Description (required), GeoJSON File (required) <br> 4. Sub-administrative zone IDs are pre-filled from source EA <br> 5. Remove button allows removing individual EA forms (minimum 2 must remain) <br> 6. Form validation ensures at least 2 new EAs are configured <br> 7. All EA forms are validated independently | P |
| TC1-804 | Create Split Enumeration Areas | Verify that admin can split a source EA into multiple new EAs successfully | 1. Source EA is selected and validated <br> 2. Minimum 2 new EA forms are configured with valid data <br> 3. Each new EA has unique name, area code, description, and GeoJSON file <br> 4. Form validation works for all required fields and formats <br> 5. GeoJSON files are validated for each new EA <br> 6. Submitting valid form splits source EA into new EAs <br> 7. Success message displays: "Enumeration area split successfully" <br> 8. Source EA is deactivated and new EAs are created <br> 9. Error messages display for validation failures or server errors | P |
| TC1-805 | View List of Split EAs | Verify that admin can view paginated list of split enumeration areas | 1. Split EAs list page loads successfully <br> 2. Table displays split EAs with columns: ID, Name, Area Code, Description, Operation Date, Actions <br> 3. Pagination controls are functional (10, 25, 50, 100 rows per page) <br> 4. Table supports sorting by operation date (default: descending) <br> 5. Search functionality filters split EAs by name, area code, or description <br> 6. Loading indicator shows during data fetch <br> 7. Error message displays if loading fails | P |
| TC1-806 | View Split History | Verify that admin can view detailed history of split operation | 1. "View History" button is visible for each split EA in the list <br> 2. Clicking "View History" opens history viewer dialog <br> 3. History displays: source EA that was split, new EAs created from split, operation date, reason (if provided) <br> 4. History information is accurate and complete <br> 5. Dialog can be closed or maximized <br> 6. Error message displays if history loading fails | P |
| TC1-807 | Search Split EAs | Verify that search functionality filters split enumeration areas correctly | 1. Search input field accepts text input <br> 2. Search filters results by name, area code, or description <br> 3. Clear search button appears when search has value <br> 4. Clearing search resets table to show all split EAs <br> 5. Search is case-insensitive and supports partial matches <br> 6. Search works with pagination | P |
| TC1-808 | Form Validation - Split Operation | Verify that all form validations work correctly in split operation | 1. Source EA selection is required <br> 2. Minimum 2 new EA forms are required <br> 3. Each new EA requires: Name (min 2 chars), Area Code (alphanumeric), Description (min 5 chars), GeoJSON File <br> 4. GeoJSON file validation works for each new EA <br> 5. Error messages display below each invalid field <br> 6. Submit button is disabled when form is invalid <br> 7. Validation prevents duplicate area codes within the split operation | P |

## Test Status Legend
- **P** = Pass (Test case is ready for execution)
- **F** = Fail (Test case needs revision or has issues)
- **B** = Blocked (Test case cannot be executed due to dependencies)
