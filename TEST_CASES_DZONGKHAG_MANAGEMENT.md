# Test Cases - Dzongkhag Management Module

## Test Case Documentation

| Test ID | Module / Feature | Objective | Expected Results | Status (P/F/B) |
|---------|------------------|-----------|------------------|----------------|
| TC1-301 | Load Dzongkhags | Verify that all dzongkhags are loaded and displayed correctly | 1. Dzongkhag Management page loads successfully <br> 2. Dzongkhag list table displays all dzongkhags with columns: ID, Name, Area Code, Actions <br> 3. Map displays all dzongkhags with geographic boundaries <br> 4. Loading indicator shows during data fetch <br> 5. Error message displays if loading fails | P |
| TC1-302 | View Dzongkhag | Verify that admin can view dzongkhag details and data | 1. Clicking "View Details" button navigates to dzongkhag data viewer page <br> 2. Clicking on map feature displays popup with dzongkhag information <br> 3. Selecting dzongkhag from table highlights it in the map <br> 4. Map popup shows dzongkhag name, area code, and download options | P |
| TC1-303 | Create Dzongkhag | Verify that admin can create a new dzongkhag successfully | 1. "Add Dzongkhag" button opens "Add New Dzongkhag" dialog <br> 2. Dialog displays form fields: Dzongkhag Name (required, min 2 chars), Area Code (required, alphanumeric) <br> 3. Form validation works for required fields and format <br> 4. Submitting valid form creates dzongkhag <br> 5. Success message displays: "Dzongkhag created successfully" <br> 6. Dialog closes and table refreshes with new dzongkhag <br> 7. Error messages display for validation failures or server errors | P |
| TC1-304 | Edit Dzongkhag | Verify that admin can update dzongkhag information | 1. Clicking "Edit" button on dzongkhag row opens "Edit Dzongkhag" dialog <br> 2. Dialog pre-fills with current dzongkhag data: Name, Area Code <br> 3. Form validation works for editable fields <br> 4. Submitting with changes updates dzongkhag successfully <br> 5. Success message displays: "Dzongkhag updated successfully" <br> 6. Dialog closes and table refreshes with updated data <br> 7. Error messages display for validation failures or server errors | P |
| TC1-305 | Delete Dzongkhag | Verify that admin can delete a dzongkhag | 1. Clicking "Delete" button on dzongkhag row opens confirmation dialog <br> 2. Confirmation dialog displays: "Are you sure you want to delete [Name]?" with warning message <br> 3. Confirmation dialog has "Cancel" and "Delete" buttons <br> 4. Clicking "Delete" removes dzongkhag <br> 5. Success message displays: "Dzongkhag deleted successfully" <br> 6. Dzongkhag is removed from table and map <br> 7. Clicking "Cancel" closes dialog without deletion <br> 8. Error messages display if deletion fails | P |
| TC1-306 | Upload GeoJSON | Verify that admin can upload GeoJSON file for a dzongkhag | 1. Clicking "Upload GeoJSON" button opens "Upload GeoJSON" dialog <br> 2. Dialog displays selected dzongkhag information and file upload field <br> 3. File upload accepts .json and .geojson files (max 10MB) <br> 4. File validation works: shows error for invalid file types <br> 5. Submitting valid file uploads GeoJSON successfully <br> 6. Success message displays: "GeoJSON uploaded successfully" <br> 7. Dialog closes and map refreshes with new boundaries <br> 8. Error messages display for invalid files or upload failures | P |
| TC1-307 | Search Dzongkhags | Verify that search functionality filters dzongkhags correctly | 1. Search input field accepts text input <br> 2. Search filters results by ID, name, or area code <br> 3. Clear search button (X) appears when search has value <br> 4. Clearing search resets table to show all dzongkhags <br> 5. Search is case-insensitive and supports partial matches | P |
| TC1-308 | Map View | Verify that map view displays correctly with all dzongkhag boundaries | 1. Map loads and displays in the right panel (50% width) <br> 2. All dzongkhag boundaries are rendered on the map with proper colors <br> 3. Map is centered on Bhutan with appropriate zoom level <br> 4. Map automatically fits bounds to show all dzongkhags <br> 5. Loading overlay displays while map data is being fetched <br> 6. Map container is properly sized and responsive | P |
| TC1-309 | Download GeoJSON from Map | Verify that admin can download GeoJSON file for a dzongkhag from map popup | 1. Clicking on a dzongkhag feature on the map opens a popup <br> 2. Popup displays dzongkhag name, area code, and download buttons <br> 3. Clicking "GeoJSON" button in popup downloads the feature as GeoJSON file <br> 4. Downloaded file has format: "[dzongkhag_name].geojson" <br> 5. Success message displays: "GeoJSON file downloaded successfully" <br> 6. File contains FeatureCollection with the selected dzongkhag feature | P |
| TC1-310 | Download KML from Map | Verify that admin can download KML file for a dzongkhag from map popup | 1. Clicking on a dzongkhag feature on the map opens a popup <br> 2. Popup displays "KML" download button <br> 3. Clicking "KML" button in popup downloads the feature as KML file <br> 4. Downloaded file has format: "[dzongkhag_name].kml" <br> 5. Success message displays: "KML file downloaded successfully" <br> 6. File contains KML format with the selected dzongkhag feature | P |
| TC1-311 | Basemap Selection | Verify that basemap selector allows switching between different map styles | 1. Basemap selector dropdown is visible in map header <br> 2. Dropdown displays basemap categories and options (e.g., Positron, OpenStreetMap, Satellite) <br> 3. Selecting a different basemap updates the map background layer <br> 4. Map boundaries remain visible after basemap change <br> 5. Selected basemap persists during the session <br> 6. Default basemap is "Positron" | P |
| TC1-312 | Map Interaction | Verify that map interactions work correctly | 1. Clicking on map feature selects corresponding dzongkhag in table <br> 2. Hovering over map features highlights them with increased opacity <br> 3. Map popup displays dzongkhag information and download options <br> 4. Tooltip shows dzongkhag name on hover <br> 5. Reload map button refreshes map data and boundaries <br> 6. Map interactions do not interfere with table selection | P |

## Test Status Legend
- **P** = Pass (Test case is ready for execution)
- **F** = Fail (Test case needs revision or has issues)
- **B** = Blocked (Test case cannot be executed due to dependencies)
