# Test Cases - Administrative Zone Management Module

## Test Case Documentation

| Test ID | Module / Feature | Objective | Expected Results | Status (P/F/B) |
|---------|------------------|-----------|------------------|----------------|
| TC1-401 | Load Administrative Zones | Verify that administrative zones are loaded and displayed correctly based on selected dzongkhag | 1. Administrative Zone Management page loads successfully <br> 2. Dzongkhag dropdown displays all available dzongkhags <br> 3. First dzongkhag is auto-selected by default <br> 4. Administrative zones table displays zones for selected dzongkhag with columns: ID, Name, Type, Area Code, Actions <br> 5. Map displays administrative zone boundaries for selected dzongkhag <br> 6. Loading indicator shows during data fetch <br> 7. Error message displays if loading fails | P |
| TC1-402 | Select Dzongkhag | Verify that selecting a dzongkhag filters and loads corresponding administrative zones | 1. Dzongkhag dropdown is visible and functional <br> 2. Selecting a different dzongkhag from dropdown updates the administrative zones list <br> 3. Map refreshes to show boundaries for selected dzongkhag <br> 4. Selected dzongkhag is saved and persists <br> 5. Administrative zone selection is reset when dzongkhag changes <br> 6. Statistics (Gewog count, Thromde count) update based on filtered zones | P |
| TC1-403 | View Administrative Zone | Verify that admin can view administrative zone details and data | 1. Clicking "View Details" button navigates to administrative zone data viewer page <br> 2. Clicking on map feature displays popup with zone information (name, type, area code) <br> 3. Selecting zone from table highlights it in the map <br> 4. Map popup shows download options (GeoJSON, KML) <br> 5. Zone labels are displayed on map with name and area code | P |
| TC1-404 | Create Administrative Zone | Verify that admin can create a new administrative zone successfully | 1. "Add Administrative Zone" button opens "Add New Administrative Zone" dialog <br> 2. Dialog displays form fields: Name (required, min 2 chars), Area Code (optional), Dzongkhag (required, pre-selected), Type (required: Gewog/Thromde) <br> 3. Form validation works for required fields and format <br> 4. Submitting valid form creates administrative zone <br> 5. Success message displays: "Administrative zone created successfully" <br> 6. Dialog closes and table refreshes with new zone <br> 7. Map refreshes to show new zone boundary <br> 8. Error messages display for validation failures or server errors | P |
| TC1-405 | Edit Administrative Zone | Verify that admin can update administrative zone information | 1. Clicking "Edit" button on zone row opens "Edit Administrative Zone" dialog <br> 2. Dialog pre-fills with current zone data: Name, Area Code, Dzongkhag, Type <br> 3. Form validation works for editable fields <br> 4. Submitting with changes updates zone successfully <br> 5. Success message displays: "Administrative zone updated successfully" <br> 6. Dialog closes and table refreshes with updated data <br> 7. Map refreshes to reflect changes <br> 8. Error messages display for validation failures or server errors | P |
| TC1-406 | Delete Administrative Zone | Verify that admin can delete an administrative zone | 1. Clicking "Delete" button on zone row opens confirmation dialog <br> 2. Confirmation dialog displays: "Are you sure you want to delete [Name]?" with warning message <br> 3. Confirmation dialog has "Cancel" and "Delete" buttons <br> 4. Clicking "Delete" removes administrative zone <br> 5. Success message displays: "Administrative zone deleted successfully" <br> 6. Zone is removed from table and map <br> 7. Clicking "Cancel" closes dialog without deletion <br> 8. Error messages display if deletion fails | P |
| TC1-407 | Upload GeoJSON | Verify that admin can upload GeoJSON file for an administrative zone | 1. Clicking "Upload GeoJSON" button opens "Upload GeoJSON" dialog <br> 2. Dialog displays selected zone information and file upload field <br> 3. File upload accepts .json and .geojson files (max 10MB) <br> 4. File validation works: shows error for invalid file types <br> 5. Submitting valid file uploads GeoJSON successfully <br> 6. Success message displays: "GeoJSON uploaded successfully" <br> 7. Dialog closes and map refreshes with new boundaries <br> 8. Error messages display for invalid files or upload failures | P |
| TC1-408 | Bulk Upload Administrative Zones | Verify that admin can bulk upload multiple administrative zones via CSV/Excel file | 1. "Bulk Upload" button opens bulk upload dialog <br> 2. Dialog requires dzongkhag selection and file upload <br> 3. File upload accepts supported formats (CSV/Excel) <br> 4. Upload process shows progress indicator <br> 5. Upload results display: success count, failure count, and error details <br> 6. Successfully uploaded zones appear in table <br> 7. Error messages display for invalid files or upload failures | P |
| TC1-409 | Search Administrative Zones | Verify that search functionality filters zones correctly | 1. Search input field accepts text input <br> 2. Search filters results by ID, name, or area code <br> 3. Clear search button (X) appears when search has value <br> 4. Clearing search resets table to show all zones for selected dzongkhag <br> 5. Search is case-insensitive and supports partial matches | P |
| TC1-410 | Map View | Verify that map view displays correctly with administrative zone boundaries | 1. Map loads and displays in the right panel <br> 2. Administrative zone boundaries are rendered on the map with proper colors (Gewog: blue, Thromde: green) <br> 3. Map is centered on selected dzongkhag with appropriate zoom level <br> 4. Zone labels display name and area code on map <br> 5. Loading overlay displays while map data is being fetched <br> 6. Map container is properly sized and responsive | P |
| TC1-411 | Download GeoJSON from Map | Verify that admin can download GeoJSON file for an administrative zone from map popup | 1. Clicking on an administrative zone feature on the map opens a popup <br> 2. Popup displays zone name, type, area code, and download buttons <br> 3. Clicking "GeoJSON" button in popup downloads the feature as GeoJSON file <br> 4. Downloaded file has format: "[zone_name].geojson" <br> 5. Success message displays: "GeoJSON file downloaded successfully" <br> 6. File contains FeatureCollection with the selected zone feature | P |
| TC1-412 | Download KML from Map | Verify that admin can download KML file for an administrative zone from map popup | 1. Clicking on an administrative zone feature on the map opens a popup <br> 2. Popup displays "KML" download button <br> 3. Clicking "KML" button in popup downloads the feature as KML file <br> 4. Downloaded file has format: "[zone_name].kml" <br> 5. Success message displays: "KML file downloaded successfully" <br> 6. File contains KML format with the selected zone feature | P |
| TC1-413 | Basemap Selection | Verify that basemap selector allows switching between different map styles | 1. Basemap selector dropdown is visible in map header <br> 2. Dropdown displays basemap categories and options (e.g., Positron, OpenStreetMap, Satellite) <br> 3. Selecting a different basemap updates the map background layer <br> 4. Zone boundaries remain visible after basemap change <br> 5. Selected basemap persists during the session <br> 6. Default basemap is "Positron" | P |
| TC1-414 | Layer Toggles | Verify that layer toggles show/hide different map layers correctly | 1. Layer toggle controls are visible (Administrative Zones, Sub-Administrative Zones, Enumeration Areas, Dzongkhag) <br> 2. Toggling "Administrative Zones" layer shows/hides zone boundaries <br> 3. Toggling "Sub-Administrative Zones" layer shows/hides SAZ boundaries (if data available) <br> 4. Toggling "Enumeration Areas" layer shows/hides EA boundaries (if data available) <br> 5. Toggling "Dzongkhag" layer shows/hides dzongkhag boundaries <br> 6. Multiple layers can be displayed simultaneously <br> 7. Layer states persist during session | P |
| TC1-415 | Map Interaction | Verify that map interactions work correctly | 1. Clicking on map feature selects corresponding zone in table <br> 2. Hovering over map features highlights them with increased opacity <br> 3. Map popup displays zone information and download options <br> 4. Tooltip shows zone name and area code on hover <br> 5. Reload map button refreshes map data and boundaries <br> 6. Map interactions do not interfere with table selection | P |

## Test Status Legend
- **P** = Pass (Test case is ready for execution)
- **F** = Fail (Test case needs revision or has issues)
- **B** = Blocked (Test case cannot be executed due to dependencies)
